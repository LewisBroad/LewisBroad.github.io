Current Project: [Untitled Coop Game]

I am currently working on a coop puzzle game for my final year project. In this project, I am focusing on developing the game mainly using C++ in Unreal Engine 5. I have created the multiplayer subsystem and I already have replicated actor components for the puzzles. Here are some key features I have planned for the project:

  • Unique Character Abilities and Traits: Each player will control one of two characters. I'm planning for the characters to have their own abilities such as being able to break glass and traits such as weight used to help them complete puzzles.
  • Random Generated Puzzles: Another feature I'm implementing is for puzzle answers to be random each time the game is played. This gives the player a slight change each time and means that the puzzles can not be immediately skipped if the player knows the answer.
  • Thematic Environment: The environment I have planned is going to tell a story. While the game can be played without a care for how the environment looks, I'd like players to be able to look around at the scenery and feel like there is a deeper story to be told.

Project 2: Baldy vs Zombie2

I made a 2D zombie shooter game in which we were graded mainly on the coding. The game uses pooling for the spawning of enemies and bullets. The game is timed and ends on the characters health hitting 0 or the time. The aim is to eliminate as many zombies as possible in the given time. An issue I had with the game was that the projectiles fired from the player would collide with the player and so sometimes would seemingly not fire. To fix this, I had a separate entity for the gun that follows the player and fires from that.

C++ Advanced Programming

I created a C++ test based game for one of my projects. Originally this was a project I started over the summer but figured I can use it as one of my university projects. This game is an adventure game with character classes that set the characters initial stats. For example the Knight class has High armour and HP but lower attack damage, whereas the rogue class is the opposite to this. The game has the player wondering around and has a random encounter chance with there currently being 2 encounters. These being an enemy will spawn from either a tank enemy or a damage enemy (I called them a magic tree and a goblin.) And a magic gnome that will buff the player or laugh and leave. This project was ultimately a skills test for my knowledge in C++ and I utilised multiple methods such as exception handling, memory management, winsock, and overloading. This project I did get carried away with because I was enjoying adding more content to the game but I had to remove some features as we were limited to a 10 class limit.

Unity Shaders

I created 4 shaders for my shaders module. We were taught different shaders and here is what I made for my assignment:

  • A good idea I had was to combine the cube map and the mirror shader I learned a tweak them a little. I added a code that detected if the player was near enough so you could see a different cube map such as if you were looking through a window and I had the "Mirror" there to just reflect the player position in the texture.
  • I made a wave shader that moved along a wave frequency with an animated texture on top to simulate the ocean.
  • I created a rusting texture that would change the roughness and glossiness of a texture over time.
  • Lastly I created a dissolving texture that took a bump map and dissolved the model according to the difference.
