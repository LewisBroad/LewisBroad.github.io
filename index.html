Unreal Engine Projects

Dissertation Project: Divided

I created a coop puzzle game for my final year project. In this project, I was focusing on developing the game mainly using C++ in Unreal Engine 5. I created the multiplayer subsystem and I had replicated actor components for the puzzles. Here are some key features in the project:

  • Unique Character Abilities and Traits: Each player controls one of two characters. I'm the characters have their own abilities such as being able to teleport, destroy walls and shoot targets which can be used to help them complete puzzles.
  • Puzzles: The aim of this game is to work together to solve puzzles and get to the end of the level. There are different ways of making it passed each obstacle and it is up to the players to figure out their own way.
  • Thematic Environment: The environment is telling a story. While the game can be played without a care for how the environment looks, I'd like players to be able to look around at the scenery and feel like there is a deeper story to be told.

This was the first external testing of the multiplayer functionality. I got two of my friends to playest the game to get a better understanding of any bugs, as well as overlooked features. I got feedback from this test finding out some overlooked features that would've helped the players understand what they had to do. This feedback resulted in me adding highlighted surfaces to make it a bit more clear on the objective.

This video shows off the first original test of my gameplay features. Including replication of moving platforms, as well as their state being altered by a replicated pressure pad

Here I demonstrate the first prototype for the teleporter character. The way this is used is that the player presses a button to activate a teleporting state and can see visually where they are teleporting to. A Boolean is used to determine whether the player was in this state or not and in the update function, it uses a line trace that is used to detect where the player is aiming at. This is the base trace for detecting the location but there is a second line trace after this that detects walls using a dot product of the location the returns. If the dot product returns under a certain amount, then the teleport is valid and the player is free to teleport to that location.
An issue that came up was that the player was teleporting through the floor because the teleport would move the players centre to the location, but this was easily fixed by adding the player capsule’s half-height to the Y-axis. Finally allowing the character to be able to teleport through certain meshes, with that being glass, this requires the need to create a new trace channel so that the trace ignored certain collision presets. One was created for the teleporting trace that was blocked by everything but on glass it ignores the collision and allows the player to teleport through.

This is a video showcasing the gun character. The bullets fired from this gun are able to interact with targets that start turning to open up a connected gate. These gates have a timer on them where if the value to fully open them is not met, the gate will start to close again.

Unity Projects

Unity Engine Roguelike

I am currently developing a roguelike game in the unity engine in my spare time. This game is very early in development but I have big plans for the future of this game. The game will feature the gameplay loop of landing on new planets, killing enemies, looting, diving deeper into the planet for better loot and harder enemies, and completing a timed extraction.

  • Character Selection Each playthrough you will be able to play as one of the selectable characters. Each will different playstyles.
  • Party System After your selected character is chosen, it will be time to choose your team from all the non-selected characters.
  • Upgrades After each extraction, aboard your ship you will be able to upgrade yourself and your party. Whether that be stat upgrades or upgrades to your abilities.
  • Different Environments There are plans to feature different planets with unique variations of enemies on them.
  • Extractions Extractions will be able to be called in towards the ending of the level. These will require time and defending to complete but once you have done this, you will leave the planet.

Project 2: Baldy vs Zombie²

I made a 2D zombie shooter game in which we were graded mainly on the coding. The game uses pooling for the spawning of enemies and bullets. The game is timed and ends on the characters health hitting 0 or the time. The aim is to eliminate as many zombies as possible in the given time. An issue I had with the game was that the projectiles fired from the player would collide with the player and so sometimes would seemingly not fire. To fix this, I had a separate entity for the gun that follows the player and fires from that.
